网络分层的真实含义是什么

2018-08-01

四个问题

  1. TCP 在进行三次握手的时候，IP 层和 MAC 层对应都有什么操作呢？

    TCP 发送每一个消息，都会带着 IP 层和 MAC 层了。因为，TCP 每发送一个消息，IP 层和 MAC 层的所有机制都要运行一遍。而你只看到 TCP 三次握手了，其实，IP 层和 MAC 层为此也忙活好久了。

  2. A 知道自己的下一个中转站是 B，那从 A 发出来的包，应该把 B 的 IP 地址放在哪里呢？B 知道自己的下一个中转站是 C，从 B 发出来的包，应该把 C 的 IP 地址放在哪里呢？如果放在 IP 协议中的目标地址，那包到了中转站，怎么知道最终的目的地址是 D 呢？

  1. 二层设备处理的通常是 MAC 层的东西。那我发送一个 HTTP 的包，是在第七层工作的，那是不是不需要经过二层设备？或者即便经过了，二层设备也不处理呢？或者换一种问法，二层设备处理的包里，有没有 HTTP 层的内容呢？

    二层设备收进去的是整个网络包。这里面 HTTP、TCP、 IP、 MAC 都有。什么叫二层设备，就是只把 MAC 头摘下来，看看到底是丢弃、转发，还是自己留着。

  2. 从你的电脑，通过 SSH 登录到公有云主机里面，都需要经历哪些过程？或者说你打开一个电商网站，都需要经历哪些过程？说得越详细越好。

网络为什么要分层？

因为复杂的程序都要分层。所谓的二层设备、三层设备，都是这些设备上跑的程序不同而已。始终想象自己是一个处理网络包的程序，如何拿到网络包、如何根据规则进行处理、如何把包发出去。

程序是如何工作的？

想象自己是一个处理网络包的程序，接收到Buffer时，调用process_layer2(buffer)函数，摘掉二层的头（MAC头），如果MAC地址和自己的相符，则说明包是发送给自己的。继续调用process_layer3(buffer)函数，此时Buffer里已经没有二层的头（MAC头）了。如果IP地址不是自己的，那就应该转发出去；如果IP地址是自己的，就是发送给自己的。摘掉三层的头（IP头），继续交给上一层处理。假设上层是基于TCP，则调用process_tcp(buffer)函数。此时Buffer中已经没有三层的头（IP头），所以需要查看四层的头（TCP头）。看这是发起还是应答，或者是一个普通的数据包。如果是发起或者应答，则接下来可能要发送一个回复包。如果是一个普通数据包，则交由上层处理（process_http(buffer)函数）。或者交由应用程序程序自己去处理，交给哪个应用程序呢？在四层的头（TCP头）里有端口号，不同的程序的监听不同的端口号。我们假设交给浏览器去处理，浏览器解析HTML，展示出页面。

此时用户点击页面上的按钮，浏览器发起一次新的HTTP请求。首先调用send_tcp(buffer)函数，buffer就是HTTP请求内容，这个函数会给buffer加一个TCP头，记录下源端口号和目标端口号（浏览器会提供）。然后调用send_layer3(buffer)函数，此时buffer中包含了HTTP的请求内容和请求头以及TCP头，在这个函数里加一个IP头，记录下源IP地址和目标IP地址。然后调用send_layer2(buffer)函数，buffer里面已经包含了HTTP的请求内容和请求头、TCP头以及IP头，在这个函数里面加一个MAC头，记录源MAC地址和目标MAC地址，此时目标MAC地址不一定能知道，如果不知道的话，需要通过ARP协议获取MAC地址。MAC地址是必须要有的。

层与层之间的关系

层层封装的关系，可以想象为套娃。只要是在网络上跑的包，都是完整的。可以有下层没上层，但是绝对不可能有上层没下层。
